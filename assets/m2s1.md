## Actividad 1. Características generales de diversos sistemas operativos y su relevancia en la seguridad de la información

| Sistema Operativo | Tipo de Kernel     | Modelo de Seguridad                                  | Actualizaciones y Parches                       | Herramientas/Built‑in Security                        | Relevancia en Seguridad de la Información                                                                  |
| ----------------- | ------------------ | ---------------------------------------------------- | ----------------------------------------------- | ----------------------------------------------------- | ---------------------------------------------------------------------------------------------------------- |
| **Windows 10/11** | Monolítico híbrido | ACLs (listas de control de acceso), UAC              | Windows Update (centralizado), WSUS en empresas | BitLocker, Windows Defender, Credential Guard         | Muy extendido en entornos corporativos; parches frecuentes; necesita buena gestión de parches y antivirus. |
| **Ubuntu LTS**    | Monolítico (Linux) | Discrecional (DAC) + opcional SELinux/AppArmor       | `apt update/upgrade`; repos oficiales y PPA     | AppArmor, firewall UFW, criptografía LUKS             | Popular en servidores y estaciones; fuerte comunidad; AppArmor simplifica confinamiento de procesos.       |
| **CentOS / RHEL** | Monolítico (Linux) | Discrecional (DAC) + SELinux obligatorio             | `yum update`; canales de pago en RHEL           | SELinux, firewall D-Bus, SCAP (compliance)            | Estándar en data centers; SELinux por defecto endurece todo el sistema; ideal para cumplimiento normativo. |
| **macOS**         | Micro-kernel XNU   | DAC + sandboxing + SIP (System Integrity Protection) | Actualizaciones vía App Store y combinadas      | FileVault, Gatekeeper, SIP, XProtect                  | Muy usado en entornos creativos; SIP y Gatekeeper limitan la ejecución de malware y protegen integridad.   |
| **FreeBSD**       | Monolítico BSD     | DAC + jails (contenedores ligeros)                   | `freebsd-update`, ports tree                    | Jails, PF firewall, Mandatory Access Control opcional | Alta estabilidad; jails ofrecen aislamiento; buena documentación de seguridad.                             |

## Actividad 2. Proceso para instalar el sistema operativo

1. **Recopilación de requerimientos**

   * Entrevista al usuario o área solicitante para definir: propósito (servidor, estación de trabajo), aplicaciones críticas, nivel de disponibilidad.
   * Lista de software asociado (antivirus, agentes de monitoreo, etc.).

2. **Verificación de compatibilidad de hardware/software**

   * Chequeo de CPU, RAM, espacio en disco, firmware (BIOS/UEFI) y versiones compatibles con el SO.
   * Validación de requisitos mínimos y recomendados según fabricante.

3. **Descarga de medios de instalación**

   * Obtener ISO oficial y firmado desde sitio del fabricante (Windows Media Creation Tool, Ubuntu LTS, etc.).
   * Verificar integridad con checksum (SHA‑256/MD5).

4. **Preparación del medio físico**

   * Crear USB booteable (Rufus, `dd`, Etcher) o grabar DVD.
   * Configurar secuencia de arranque en BIOS/UEFI.

5. **Instalación paso a paso**

   * Iniciar desde medio y seguir asistente de instalación:

     1. Selección de idioma y zona horaria.
     2. Particionado de disco (ext4, NTFS, APFS, LVM según necesidad).
     3. Configuración de red (DHCP vs estática).
     4. Creación de usuario administrador inicial.
     5. Elección de telemetría y políticas de privacidad.
   * Aplicar parámetros recomendados por fabricante (p. ej., ZTP en Windows, LVM en Linux).

6. **Configuraciones post‑instalación**

   * Ajustes de BIOS/UEFI (seguridad de arranque, TPM, Secure Boot).
   * Creación de snapshots (partición de recuperación) si aplica.

> **Colaboración y revisión:**
>
> * Documentar cada paso en un bitácora compartida.
> * Revisar con tu jefe inmediato la configuración antes de avanzar a producción.

## Actividad 3. Instalación de software de terceros (drivers) y servicios de seguridad

1. **Inventario de drivers necesarios**

   * Identificar dispositivos críticos (red, GPU, chipset) y versiones compatibles.
   * Descargar controladores firmados desde el sitio del fabricante.

2. **Proceso de instalación de drivers**

   * Ejecutar instaladores en orden recomendado (chipset → tarjeta de red → GPU → periféricos).
   * Reinicios entre instalaciones si el sistema lo solicita.

3. **Verificación y validación**

   * En Windows: revisar en “Administrador de dispositivos” que no haya errores.
   * En Linux: usar `lspci`, `lsmod`, `dmesg` para confirmar carga correcta.

4. **Configuración de servicios de seguridad del SO**

   * Encender y configurar:

     * Antivirus en tiempo real (Windows Defender, ClamAV).
     * Firewall (Windows Firewall, iptables/UFW, PF).
     * BitLocker/FileVault/LUKS para cifrado de disco.
   * Ajustar políticas de grupo (GPO) o perfiles de seguridad (SCAP, Bastille).

5. **Pruebas de resistencia**

   * Simular ataques básicos (escaneo de puertos, ejecución de software no firmado).
   * Corroborar que el SO bloquea o registra adecuadamente.

> **Trabajo en equipo:**
>
> * Asignar roles (quién instala, quién prueba, quién documenta).
> * Reportar progreso diariamente a tu jefe y ajustar prioridades.

## Actividad 4. Agregar usuarios en el sistema operativo

1. **Definición de perfiles y jerarquías**

   * Crear un organigrama con niveles de acceso (administrador, power user, usuario estándar).
   * Definir roles y grupos (ej. Finanzas, TI, Ventas).

2. **Creación de cuentas**

   * **Windows:**

     ```powershell
     New-LocalUser -Name "usuario" -FullName "Nombre Completo" -Description "Rol" -Password (ConvertTo-SecureString "P@ssw0rd" -AsPlainText -Force)
     Add-LocalGroupMember -Group "Grupo" -Member "usuario"
     ```
   * **Linux:**

     ```bash
     sudo useradd -m -G grupo1,grupo2 -s /bin/bash usuario
     sudo passwd usuario
     ```

3. **Políticas de contraseñas y expiración**

   * Establecer longitud mínima, complejidad y caducidad.
   * En Linux: editar `/etc/login.defs` y aplicar `chage`.

4. **Auditoría y registro**

   * Habilitar Syslog/Windows Event Log para eventos de gestión de usuarios.
   * Configurar alertas (SIEM) en caso de creación o cambio de cuentas privilegiadas.

> **Coordinación:**
>
> * Validar con Recursos Humanos y con el jefe inmediato la lista de usuarios a crear.
> * Registrar cada alta en la bitácora compartida.

## Actividad 5. Generar evidencia digital y documentación

1. **Recopilación de registros**

   * Exportar lista de usuarios, grupos y permisos:

     * Windows: `Get-LocalUser` y `Get-LocalGroupMember` en PowerShell.
     * Linux: `/etc/passwd`, `/etc/group`, `getent shadow`.

2. **Formato de evidencia**

   * Plantilla estándar con:

     * Fecha y hora de extracción.
     * Nombre del sistema y versión.
     * Responsable de la extracción.
     * Tabla de usuarios con jerarquía y responsabilidades.

3. **Cumplimiento normativo**

   * Incluir cláusulas legales aplicables (Ley Federal de Protección de Datos Personales en Posesión de los Particulares).
   * Señalar medidas de resguardo (cifrado de evidencias, control de acceso al repositorio).

4. **Redacción y buenas prácticas**

   * Uso de lenguaje claro y conciso.
   * Numerar secciones, encabezados consistentes.
   * Incluir firmas electrónicas o sellos de tiempo si corresponde.

5. **Almacenamiento y colaboración**

   * Guardar en repositorio corporativo (SharePoint, Git interno) con control de versiones.
   * Compartir con jefe inmediato y área de Auditoría mediante ticket o correo institucional.

> **Notas finales:**
>
> * Mantén comunicación constante con tu jefe para validar contenido y formato.
> * Trabaja en equipo: revisiones cruzadas y aprobaciones formales antes de archivar.
