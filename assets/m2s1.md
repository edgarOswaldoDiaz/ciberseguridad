Aquí tienes la tabla solicitada, adaptada para los sistemas operativos **Ms. Windows Server** y **Ubuntu 24.04 LTS**, que resume los pasos para instalar software de terceros (drivers) y configurar servicios de seguridad. La tabla incluye los pasos clave, sus adaptaciones específicas para cada sistema operativo y observaciones adicionales cuando es necesario.

| **Paso**                  | **Ms. Windows Server**                                                                 | **Ubuntu 24.04 LTS**                                                                  | **Observaciones**                                                                 |

| **1. ### Notas Adicionales
- **Diferencias entre Sistemas**: Windows Server utiliza interfaces gráficas para muchas tareas (como el "Administrador de dispositivos" o "Seguridad de Windows"), mientras que Ubuntu 24.04 LTS depende principalmente de comandos en la terminal.
- **Seguridad**: Ambos sistemas requieren configurar un firewall y, opcionalmente, un antivirus, pero las herramientas específicas varían (Windows Defender vs. ClamAV, por ejemplo).
- **Mantenimiento**: Establecer un plan de actualizaciones es clave para garantizar la estabilidad y seguridad a largo plazo.


## Agregar usuarios en el sistema operativo, atendiendo al nivel jerárquico y responsabilidades dentro de la organización; trabajando de forma autónoma y colaborativa, atendiendo las recomendaciones de su jefe inmediato.!

1. **Definición de perfiles y jerarquías**

   * Crear un organigrama con niveles de acceso (administrador, power user, usuario estándar).
   * Definir roles y grupos (ej. Finanzas, TI, Ventas).

2. **Creación de cuentas**

   * **Windows:**

     ```powershell
     New-LocalUser -Name "usuario" -FullName "Nombre Completo" -Description "Rol" -Password (ConvertTo-SecureString "P@ssw0rd" -AsPlainText -Force)
     Add-LocalGroupMember -Group "Grupo" -Member "usuario"
     ```
   * **Linux:**

     ```bash
     sudo useradd -m -G grupo1,grupo2 -s /bin/bash usuario
     sudo passwd usuario
     ```

3. **Políticas de contraseñas y expiración**

   * Establecer longitud mínima, complejidad y caducidad.
   * En Linux: editar `/etc/login.defs` y aplicar `chage`.

4. **Auditoría y registro**

   * Habilitar Syslog/Windows Event Log para eventos de gestión de usuarios.
   * Configurar alertas (SIEM) en caso de creación o cambio de cuentas privilegiadas.

> **Coordinación:**
>
> * Validar con Recursos Humanos y con el jefe inmediato la lista de usuarios a crear.
> * Registrar cada alta en la bitácora compartida.

## Generación de evidencia digital documentando el registro de usuarios, jerarquía y responsabilidades dentro de la organización; considerando la normatividad y legislación vigente en el tratamiento y resguardo de datos, aplicando las buenas prácticas de redacción, utilizando las tecnologías de información y comunicación, trabajando de forma autónoma y colaborativa, atendiendo las recomendaciones de su jefe inmediato.


1. **Recopilación de registros**

   * Exportar lista de usuarios, grupos y permisos:

     * Windows: `Get-LocalUser` y `Get-LocalGroupMember` en PowerShell.
     * Linux: `/etc/passwd`, `/etc/group`, `getent shadow`.

2. **Formato de evidencia**

   * Plantilla estándar con:

     * Fecha y hora de extracción.
     * Nombre del sistema y versión.
     * Responsable de la extracción.
     * Tabla de usuarios con jerarquía y responsabilidades.

3. **Cumplimiento normativo**

   * Incluir cláusulas legales aplicables (Ley Federal de Protección de Datos Personales en Posesión de los Particulares).
   * Señalar medidas de resguardo (cifrado de evidencias, control de acceso al repositorio).

4. **Redacción y buenas prácticas**

   * Uso de lenguaje claro y conciso.
   * Numerar secciones, encabezados consistentes.
   * Incluir firmas electrónicas o sellos de tiempo si corresponde.

5. **Almacenamiento y colaboración**

   * Guardar en repositorio corporativo (SharePoint, Git interno) con control de versiones.
   * Compartir con jefe inmediato y área de Auditoría mediante ticket o correo institucional.

> **Notas finales:**
>
> * Mantén comunicación constante con tu jefe para validar contenido y formato.
> * Trabaja en equipo: revisiones cruzadas y aprobaciones formales antes de archivar.
